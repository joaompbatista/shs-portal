📋 Project Context

The Scaling Smart Health Systems (SHS) project, launched under UNDP's Large Project Development Facility (LPDF) in 2025, integrates climate-smart infrastructure, digital health solutions, and innovative financing to build resilient health systems in at least 20 countries by 2029.

This data platform was built as a portfolio management tool to consolidate and visualise pipeline and project data, performance indicators and progress metrics, directly supporting the SHS Global Fund Partnership and Health Systems Team (GFPHST) at UNDP BPPS.

🔐 Authentication Flow

1
User opens the portalClicks "Sign in" in the top bar . MSAL.js initiates the Azure AD OAuth2 PKCE flow.
2
Microsoft loginUser authenticates with their Microsoft 365 / UNDP account credentials.
3
Token issuedAzure AD returns an access token scoped to Power BI Service and Fabric SQL endpoint.
4
Data access unlockedThe app uses the Bearer token in all API calls, reads and writes go directly to the Fabric Warehouse.
5
Audit trailEvery write operation (insert or update) automatically creates a record in AUDIT_LOG.

🗄️ Tables (9)

All tables in the SHS semantic model on the Fabric Warehouse.

Type Legend

Dimension Reference / dimension tables   Fact Fact / transactional tables   Audit Change history   Auxiliary Support / configuration
Table Type Cols Description / Content
DIM_COUNTRYDimension6Country dimension table. Includes ISO code, name, UNDP region and sub-region. Used for geographic analysis and country/region filtering across all reports.
DIM_DATEAuxiliary7Date table. Supports time-based analysis (year, month, quarter, day of week). Includes model last-refresh column.
DIM_GRANT_TYPEDimension4Lookup table with available grant types (e.g. Global Fund HIV, GAVI, Pandemic Fund). Includes donor name and category. Used to segment projects and pipeline opportunities by funding type.
DIM_COMPONENTDimension4Lookup table with SHS project components (e.g. Solar, Digital Health, LMIS, Climate Resilience). Used to segment KPIs and infrastructure by intervention area.
DIM_PROJECTDimension13Central table with all SHS projects. Includes country, grant type, start and end dates, phase, status, budget, disbursed amount, principal recipient and Country Office focal point. The basis for most DAX measures.
FACT_KPI_RESULTSFact11Record of KPI measurements by project and period. Includes KPI name, target value, actual value, achievement percentage, unit of measure and data quality flag. Updated via the Data Entry App.
FACT_INFRASTRUCTUREFact13Record of health infrastructure and SHS interventions. Includes facility name and type, intervention type, status, solar capacity (kW), estimated beneficiaries and GPS coordinates for mapping.
FACT_PIPELINEFact10Record of grant pipeline opportunities. Includes country, grant type, opportunity name, pipeline stage, estimated budget, priority, CO focal point, submission deadline and last contact date.
AUDIT_LOGAudit9Change history à tabela DIM_PROJECT. Cada linha regista o campo alterado, o valor anterior, o valor novo, o utilizador responsável, a data e a origem da alteração (Manual, System, Import). Preenchido automaticamente pela Data Entry App.

📐 DAX Measures (16)

All 16 measures with display folder, DAX expression and detailed description.

Measure Name Display Folder DAX Expression Description
Total Budget (€)📁 PortfolioSUM(DIM_PROJECT[budget_eur])Total approved budget across all filtered projects.
Total Disbursed (€)📁 PortfolioSUM(DIM_PROJECT[disbursed_eur])Total amount disbursed to date.
Disbursement Rate %📁 PortfolioDIVIDE([Total Disbursed (€)], [Total Budget (€)], 0)Execution rate: disbursed / approved budget.
# Projects📁 PortfolioCOUNTROWS(DIM_PROJECT)Count of projects in the current filter context.
# Countries📁 PortfolioDISTINCTCOUNT(DIM_PROJECT[country_code])Number of distinct countries with active projects.
# Projects At Risk📁 PortfolioCALCULATE(COUNTROWS(DIM_PROJECT), DIM_PROJECT[status] IN {"At Risk","Delayed","Suspended"})Projects with a risk status requiring attention.
Budget Remaining (€)📁 Portfolio[Total Budget (€)] - [Total Disbursed (€)]Undisbursed budget balance.
Avg Achievement %📁 KPI PerformanceAVERAGEX(FACT_KPI_RESULTS, DIVIDE(actual_value, target_value, 0) * 100)Average KPI achievement percentage in the filtered context.
# KPIs On Target📁 KPI PerformanceCALCULATE(COUNTROWS(FACT_KPI_RESULTS), DIVIDE(actual_value, target_value, 0) >= 0.9)KPIs achieving at least 90% of their target value.
# KPIs Flagged📁 KPI PerformanceCALCULATE(COUNTROWS(FACT_KPI_RESULTS), FACT_KPI_RESULTS[data_quality_flag] = 1)KPI records flagged for data quality review.
# Facilities📁 InfrastructureCOUNTROWS(FACT_INFRASTRUCTURE)Total health facilities in scope.
# Facilities Operational📁 InfrastructureCALCULATE(COUNTROWS(FACT_INFRASTRUCTURE), status = "Operational")Facilities with operational status.
Total Beneficiaries📁 InfrastructureSUM(FACT_INFRASTRUCTURE[beneficiaries])Total estimated beneficiaries across all health facilities.
# Solar Installations📁 InfrastructureCALCULATE(COUNTROWS(FACT_INFRASTRUCTURE), intervention_type = "Solar Panel")Count of solar panel installations.
# Pipeline Opportunities📁 PipelineCOUNTROWS(FACT_PIPELINE)Total grant pipeline opportunities tracked.
Pipeline Est. Budget (€)📁 PipelineSUM(FACT_PIPELINE[estimated_budget_eur])Total estimated value of the grant pipeline.
# High Priority Pipeline📁 PipelineCALCULATE(COUNTROWS(FACT_PIPELINE), priority = "High")High-priority pipeline opportunities.
# Pipeline Ready to Submit📁 PipelineCALCULATE(COUNTROWS(FACT_PIPELINE), stage IN {"Concept Note","Proposal","Under Review"})Opportunities at advanced pipeline stages ready for submission.

✏️ Data Entry App

Insert and edit records directly in the Fabric Warehouse. Requires Microsoft sign-in.

🚧
In development. The current version supports reading data from the semantic model (tables and dropdowns load correctly). Write operations (insert/update) to the Fabric Warehouse are not yet functional via browser. Direct T-SQL DML through the Fabric REST API is not supported from a web app due to CORS restrictions. A solution via Power Automate or Power Apps is planned.
🔐

Sign in required

The Data Entry App requires authentication with your Microsoft account to read and write data in the Fabric Warehouse.

ℹ️
Sign in using the button in the top bar to activate data entry. All changes are written directly to the Fabric Warehouse and logged in AUDIT_LOG automatically.
