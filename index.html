📋 Project Context

The Scaling Smart Health Systems (SHS) project, launched under UNDP's Large Project Development Facility (LPDF) in 2025, integrates climate-smart infrastructure, digital health solutions, and innovative financing to build resilient health systems in at least 20 countries by 2029.

This data platform was built as a portfolio management tool to consolidate and visualise pipeline and project data, performance indicators, and progress metrics — directly supporting the SHS Global Fund Partnership and Health Systems Team (GFPHST) at UNDP BPPS.

User opens the portalClicks "Sign in" in the top bar — MSAL.js initiates the Azure AD OAuth2 PKCE flow.
2
Microsoft loginUser authenticates with their Microsoft 365 / UNDP account credentials.
3
Token issuedAzure AD returns an access token scoped to Power BI Service and Fabric SQL endpoint.

Data access unlockedThe app uses the Bearer token in all API calls — reads and writes go directly to the Fabric Warehouse.

🗄️ Tables (9)

All tables in the SHS semantic model on Fabric Warehouse.

Legenda de Tipos

Dimensão Tabelas de referência / dimensão   Facto Tabelas de factos / transacionais   Auditoria Histórico de alterações   Auxiliar Suporte / configuração
Tabela Tipo Cols Descrição / Conteúdo
DIM_COUNTRYDimensão6Tabela de dimensão de países. Inclui código ISO, nome, região UNDP e sub-região. Usada para análise geográfica e filtragem por país/região em todos os relatórios.
DIM_DATEAuxiliar7Tabela de datas. Suporta análises temporais (ano, mês, trimestre, dia da semana). Inclui coluna de última atualização do modelo.
DIM_GRANT_TYPEDimensão4Lookup com os tipos de grant disponíveis (ex: Global Fund HIV, GAVI, Pandemic Fund). Inclui nome e categoria do doador. Usada para segmentar projetos e oportunidades de pipeline por tipo de financiamento.
DIM_COMPONENTDimensão4Lookup com os componentes do projeto SHS (ex: Solar, Digital Health, LMIS, Climate Resilience). Usada para segmentar KPIs e infraestruturas por área de intervenção.
DIM_PROJECTDimensão13Tabela central com todos os projetos SHS. Inclui país, tipo de grant, datas de início e fim, fase, estado, orçamento, valor executado, receptor principal e ponto focal do Country Office. Base da maioria das medidas DAX.
FACT_KPI_RESULTSFacto11Registo das medições de KPIs por projeto e período. Inclui nome do KPI, valor alvo, valor real, percentagem de cumprimento, unidade de medida e flag de qualidade de dados. Atualizado via Data Entry App.
FACT_INFRASTRUCTUREFacto13Registo das infraestruturas de saúde e intervenções SHS. Inclui nome e tipo de unidade de saúde, tipo de intervenção, estado, capacidade solar (kW), beneficiários estimados e coordenadas GPS para mapeamento.
FACT_PIPELINEFacto10Registo das oportunidades de grant em pipeline. Inclui país, tipo de grant, nome da oportunidade, fase de pipeline, orçamento estimado, prioridade, ponto focal do CO, prazo de submissão e data de último contacto.
AUDIT_LOGAuditoria9Histórico de alterações à tabela DIM_PROJECT. Cada linha regista o campo alterado, o valor anterior, o valor novo, o utilizador responsável, a data e a origem da alteração (Manual, System, Import). Preenchido automaticamente pela Data Entry App.

🔧 Fabric Warehouse — SQL Compatibility Notes

Limitations and workarounds discovered during development of this project.

⚠️
These notes are specific to Microsoft Fabric Warehouse (Synapse SQL engine) as of early 2025. Some limitations may be resolved in future Fabric updates.
StatusFeatureDetail / Workaround
✗ NoPRIMARY KEY / FOREIGN KEY constraintsNot supported in CREATE TABLE. Relationships are managed in the Power BI semantic model layer instead.
✗ NoDEFAULT values inline in CREATE TABLEcolumn INT NOT NULL DEFAULT 0 fails. Omit DEFAULT and always supply values explicitly in INSERT statements.
✗ NoComputed columnscol AS (expression) syntax not supported. Calculate derived values as DAX measures in Power BI instead.
✗ NoRecursive CTEsWITH cte AS (... UNION ALL ... FROM cte) not supported. Use pre-generated INSERT VALUES or a Python notebook for bulk data generation.
✗ NoOPTION (MAXRECURSION)Query hint not recognised. Moot since recursive CTEs are unsupported anyway.
✗ NoCREATE INDEXManual index creation not supported. The Fabric columnar engine handles query optimisation automatically.
⚠ LimitINSERT VALUES row limitMaximum 1,000 rows per INSERT VALUES statement. Split large inserts into blocks of ≤1,000 rows.
⚠ SlowWHILE loop INSERT (row-by-row)Inserting 1,826 date rows via WHILE loop took ~10 minutes due to per-transaction overhead. Use pre-generated multi-row INSERT VALUES instead — runs in seconds.
⚠ LimitMoving items between workspacesWarehouses cannot be moved between Fabric workspaces. Must be recreated. Keep all artefacts in the same workspace from the start.
⚠ NoteDirect Lake requires premium workspaceDirect Lake connections only work from workspaces assigned to a Fabric capacity. "My Workspace" (personal) is not eligible — use a dedicated workspace with Fabric trial capacity.
✓ YesBIT data typeSupported. Use 1/0 values in INSERT statements — no boolean literals.
✓ Yessp_renameSupported for renaming tables and columns: EXEC sp_rename 'table.col', 'new_name', 'COLUMN'
✓ YesALTER TABLE ADD COLUMNSupported. New columns are nullable by default.
✓ YesSELECT, INSERT, UPDATE, DELETEFull DML supported as expected.
